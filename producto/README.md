# Producto

## Objetivo

La carpeta **Producto** reúne la documentación estratégica de Flipa y constituye el punto de partida para comprender el producto desde una perspectiva funcional y de negocio.

Su propósito es documentar la visión del producto, el problema que resuelve, el alcance definido para el MVP, la evolución esperada y los objetivos que orientan el desarrollo de la plataforma.

Esta documentación permite alinear a los equipos de Producto, Negocio, Diseño, Tecnología y Operaciones sobre una misma visión, facilitando la toma de decisiones y reduciendo la dependencia del conocimiento tácito.

---

# Alcance

La carpeta Producto documenta la definición del producto antes de entrar al detalle funcional y técnico.

Incluye la información relacionada con:

- Propósito del producto.
- Visión de largo plazo.
- Problema que busca resolver.
- Objetivos estratégicos.
- Alcance del MVP.
- Funcionalidades incluidas y excluidas.
- Evolución del producto mediante el Roadmap.

Esta información sirve como insumo para las demás capas de la biblioteca de conocimiento.

| Capa | Relación |
|-------|----------|
| Negocio | Utiliza la visión del producto para definir el modelo comercial, segmentos de clientes y propuesta de valor. |
| Funcional | Toma el alcance del producto para construir requerimientos, reglas de negocio, historias de usuario y procesos. |
| Tecnología | Implementa técnicamente las funcionalidades definidas en el alcance funcional. |
| Cartera y Cobranza | Define posteriormente los procesos financieros derivados del producto. |
| Conocimiento | Facilita el onboarding y la transferencia de conocimiento utilizando la información consolidada del producto. |

No hace parte de esta carpeta la documentación técnica, arquitectura, APIs, infraestructura o implementación.

# Contexto del producto

Flipa es una plataforma orientada al sector B2B que busca digitalizar y optimizar el proceso mediante el cual las empresas ofrecen y administran soluciones financieras para sus clientes.

El producto centraliza la interacción entre los diferentes actores involucrados, permitiendo gestionar el ciclo de vida de las solicitudes, facilitar la evaluación de los usuarios y soportar los procesos operativos necesarios para la operación del negocio.

Durante el MVP el enfoque estará en validar el flujo principal del negocio y entregar las capacidades mínimas necesarias para operar el producto de forma controlada.

---

# Objetivos del producto

| Objetivo | Descripción |
|----------|-------------|
| Centralizar la operación | Consolidar en una sola plataforma la operación principal del negocio. |
| Reducir tiempos operativos | Disminuir actividades manuales durante el proceso de atención y gestión. |
| Mejorar la trazabilidad | Registrar cada etapa del proceso para facilitar seguimiento y auditoría. |
| Estandarizar procesos | Definir un flujo único para la gestión de las operaciones del producto. |
| Facilitar el crecimiento | Construir una base funcional que permita incorporar nuevos módulos sin afectar la operación existente. |

---

# Principios del producto

La documentación de Producto debe mantenerse bajo los siguientes principios:

- Una única fuente de verdad.
- Basada en decisiones de negocio.
- Fácil de entender por perfiles técnicos y no técnicos.
- Actualizada conforme evoluciona el producto.
- Versionada mediante Git.
